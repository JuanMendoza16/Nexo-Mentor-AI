Arquitectura y patrones

Arquitectura

  • State: controla estados (Idle, DatasetLoaded, Preprocessed, Training, Finished).
  • Observer: las métricas se actualizan cuando el modelo “avanza”.
  • Command: cada botón ejecuta una acción encapsulada.
  • Factory: crea datasets y trayectorias de métricas sintéticas.

Calidad y Escalabilidad

  • Código modular y mantenible.
  • Fácil de añadir nuevas etapas/métricas.
  • UI responsive con feedback inmediato.

Notas educativas

La simulación evita el coste real de entrenar un LLM, pero conserva la lógica conceptual: preparación de datos, configuración del entrenamiento y análisis de resultados.

El comportamiento de las curvas es sintético y busca ilustrar tendencias típicas: la loss desciende y la accuracy asciende hasta estabilizarse.
